# Player

Player is a self-hosted web application for browsing and watching video content provided by user-installed addons. It is designed to run on TrueNAS Scale using Docker.

## Important legal note

This project does not include any media sources, copyrighted content, scrapers, or piracy addons. Addons are user-provided integrations. The server owner is responsible for installing only addons and streams they are legally allowed to use.

## Features planned

- First-run admin registration, then admin login only.
- Guest login using a 6-digit code generated by the admin.
- Guest playback limits.
- Admin panel for logs, settings, addons, guest codes and limits.
- Addon-based catalog libraries, search, metadata, posters, backdrops, subtitles and streams.
- Netflix-like dark interface.
- Custom player UI with stream selection, speed control, fullscreen and intro skip metadata support.
- TrueNAS Scale friendly Docker deployment.

## Initial stack

- Frontend: React + Vite + Nginx
- Backend: FastAPI + SQLite
- Auth: JWT tokens
- Deployment: Docker Compose

## Quick start

```bash
cp .env.example .env
docker compose up -d --build
```

Then open:

```text
http://YOUR_TRUENAS_IP:8080
```

On the first launch, create the admin account. After that, the registration endpoint is disabled unless the database is reset.

## Addon contract draft

An addon is installed by URL. The backend expects a manifest similar to:

```json
{
  "id": "example.addon",
  "name": "Example Addon",
  "version": "1.0.0",
  "catalogs": [
    { "id": "proponowane", "name": "proponowane", "type": "movie" },
    { "id": "seriale", "name": "Seriale", "type": "series" }
  ],
  "resources": ["catalog", "meta", "stream", "subtitles"]
}
```

Expected endpoints are inspired by Stremio-like addons:

```text
GET /manifest.json
GET /catalog/{type}/{id}.json
GET /meta/{type}/{id}.json
GET /stream/{type}/{id}.json
GET /subtitles/{type}/{id}.json
```

The exact contract will be expanded during development.
